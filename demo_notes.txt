Demo Notes for Project 1

1. Introduction
- "Hi, this is a demo of my professor review QA system for Princeton CS.
- The system turns RateMyProfessors-style review text into a searchable answer engine."

2. Pipeline overview
- "There are three main stages:
  1. Ingest: `ingest.py` cleans raw review text and chunks it into short passages.
  2. Embed: `embed.py` converts chunks into embeddings with `all-MiniLM-L6-v2` and stores them in Chroma.
  3. Generate: `generator.py` retrieves relevant chunks for a query and uses Groq to produce a grounded answer."
- "I added two key improvements:
  1. Professor-specific retrieval: questions that mention a professor preferentially search that professor's review chunks.
  2. Grading-keyword filter: when a question asks about grading, the system checks if retrieved chunks mention grading-related terms (grade, strict, rigorous, fair, etc.). If not, it returns 'I don't have enough information' rather than hallucinating an answer."
